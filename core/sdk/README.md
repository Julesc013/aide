# Core SDK

## Purpose

SDK is the future stable consumption surface for authored integrations and downstream use of AIDE Core.

## Future Surfaces

- host client APIs
- service APIs
- protocol bindings
- generated artifact consumers
- host capability descriptors
- patch/approval/replay schemas

## Boundary

Q02 creates the SDK skeleton only. SDK is planned, not implemented. SDK work depends on reviewed Contract and Harness evidence and must not absorb Runtime, Bridges, host shells, or provider logic.
