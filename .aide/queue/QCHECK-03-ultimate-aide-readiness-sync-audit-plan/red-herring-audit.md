# Red-Herring Audit

Do not build these yet:

- Live Gateway/provider routing.
- Provider/model calls.
- Exact provider billing.
- Exact tokenizer integration.
- Commander/UI/mobile.
- MCP/A2A.
- Branch creation, merge, push, prune, or promotion.
- GitHub branch protection application.
- Active CI workflow installation.
- GitHub Releases.
- Git tags.
- Package publishing.
- Broad root moves.
- XStack/AuditX/RepoX/TestX deletion or rename.
- Eureka product feature work.
- Dominium product feature work.
- Doctrine rewrite.
- Automatic GPT review.
- Autonomous worker execution.

Reason: these either change trust boundaries, require target-specific review,
need install/rollback scaffolding first, or would confuse governance readiness
with product/runtime readiness.
