# Q41 Existing Tool Absorption v0 ExecPlan

## Objective

Implement deterministic, repo-local infrastructure for discovering existing
management tools, classifying their capabilities and risks, and producing
no-execution wrap plans.

## Scope

Q41 may add tool absorption policies, schemas, AIDE Lite `tools` commands,
golden tasks, tests, docs, generated advisory tool outputs, export-pack
support, and queue evidence. Q41 must not execute unknown tools, rename tools,
delete tools, migrate tools, mutate target repositories, or apply wrappers.

## Dependencies

- Q36 intent compiler and prompt normalization.
- Q37 repo intelligence outputs.
- Q38 file quality ledger outputs.
- Q39 refactor control no-apply model.
- Q40 root recycling no-apply model.

## Milestones

1. Create the Q41 queue packet and evidence scaffold.
2. Define tool absorption, inventory, fate, wrapping, risk, and capability
   policies plus portable schemas.
3. Add deterministic AIDE Lite `tools` commands for inventory, classification,
   wrap planning, validation, status, explanation, and capability summaries.
4. Add golden tasks and unittest coverage for classification and no-execution
   gates.
5. Update documentation and generated Q41 artifacts.
6. Export portable tool absorption support and write final evidence.

## Verification Intent

Run AIDE Lite validation, tests, evals, tool commands, export-pack checks, pack
status, relevant core tests, and a targeted secret scan. Record any validation
that cannot run or remains blocked.

## Recovery Notes

All generated tool outputs are advisory evidence and may be regenerated by
`py -3 .aide/scripts/aide_lite.py tools inventory`, `tools classify`, and
`tools wrap-plan`. No source tool should be modified by those commands.
