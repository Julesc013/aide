# Red-Herring and Deferred Work Audit

The following work remains explicitly deferred and must not be started by QCHECK-04 or Q49 preflight:

- install apply mode
- upgrade apply mode
- repair apply mode
- rollback apply mode
- uninstall apply mode
- active GitHub release publication
- Git tag creation or tag push
- artifact upload
- package publishing
- active CI installation
- branch protection application
- GitHub settings mutation
- model router or Gateway provider forwarding
- provider/model calls
- local model orchestration/download/setup
- MCP/A2A implementation
- Commander/UI/mobile implementation
- IDE extensions
- target repo product work before Q49/Q54 authorization
- broad root moves/deletes
- path alias application
- reference rewrite application
- target tool migration, rename, deletion, or wrapper execution

These are not QCHECK blockers. They are intentionally deferred until reviewed queue items authorize them.
