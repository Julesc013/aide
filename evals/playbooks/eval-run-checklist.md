# Eval Run Checklist

## Select Eval Scope

- Identify the eval id, lane, family, or subsystem being evaluated.
- Confirm the expected capability target and verification category.

## Select Expected Evidence

- Record what evidence the run is expected to produce.
- Record whether the run is structural, smoke-oriented, packaging-oriented, or archival.

## Run The Eval Or Verification

- Execute the defined procedure or perform the defined review.
- Record actual commands, reviewed artifacts, or referenced evidence.

## Record Result Status

- Assign `passed`, `failed`, `blocked`, `deferred`, `not-applicable`, or `unverifiable` based on real evidence.
- Keep absence of implementation distinct from failure of implementation.

## Record Blockers Or Uncertainty

- Capture blocker type, summary, and unblock conditions where relevant.
- Capture uncertainty explicitly rather than compressing it into a passing result.

## Update Documentation And Matrices

- Update catalogs, reports, or matrices only when the result materially changes repository posture.
- Keep audit trails reconstructable.
