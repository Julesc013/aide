# AIDE Labs

`labs/` tracks active experimentation, blocked work, and archival records that are not yet standardized enough to live only in the stable environment control plane.

This area differs from:

- `environments/`, which tracks concrete environment definitions, media references, snapshots, and repeatable bring-up structure
- `research/`, which tracks source-backed ecosystem facts
- `hosts/` and `shared/`, which will hold stable implementation work once experimental findings are promoted

Labs are where uncertain or partial work lives first. Successful findings should later be promoted into durable catalogs, playbooks, architecture docs, or implementation prompts.
