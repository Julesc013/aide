# Documentation Synchronization Checklist

Use this checklist after a phase that changes repo structure, implementation posture, or contributor workflow.

## Root Doc Currentness

- [ ] `README.md` reflects the current state, not an earlier bootstrap phase
- [ ] `CONTRIBUTING.md` still points contributors to the right directories and control-plane files
- [ ] `ROADMAP.md` still separates committed work from candidate or future research
- [ ] `MAINTENANCE.md` still matches actual upkeep practices
- [ ] `CHANGELOG.md` remains a baseline or is updated only with real entries

## Documentation Index Alignment

- [ ] `DOCUMENTATION.md` includes newly authoritative docs
- [ ] new reports, maintenance files, or repo-local guidance are indexed if they are meant to be authoritative

## Placeholder And Drift Checks

- [ ] stale bootstrap or placeholder language has been removed where phases have advanced
- [ ] blocked or deferred work is not hidden by polished summaries
- [ ] repo maps and current-state summaries match actual directories and evidence
