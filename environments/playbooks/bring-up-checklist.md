# Bring-Up Checklist

## Acquisition

- identify required media, toolchains, and environment family
- record source class and redistribution posture
- create catalog references before attempting bring-up

## Checksum and provenance capture

- record checksum status if hashes can be captured
- record provenance status and supporting notes
- record local storage expectation without checking assets into Git

## Install steps

- record host environment type and OS-family context
- record major install or reconstruction steps
- note where the process diverges from a reusable family pattern

## First boot

- record whether the environment reaches `boots`
- record any immediate blockers or instability
- attach provisional notes to the environment instance

## Toolchain validation

- verify required toolchain presence at a structural level
- record missing or blocked toolchain dependencies explicitly

## Snapshot creation

- capture or register a snapshot when doing so is appropriate
- record snapshot type and storage expectation

## Blocker capture

- create or update lab blocked records for unresolved issues
- distinguish acquisition blockers from boot blockers and validation blockers

## Documentation update

- update environment catalogs
- update lab registers if work is experimental or blocked
- promote durable findings into authoritative docs when the environment becomes stable
