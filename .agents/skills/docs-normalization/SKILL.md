---
name: aide-docs-normalization
description: Use when tightening AIDE documentation after planning, research, architecture, or implementation changes so root docs, indexes, and summaries match actual repository state without adding hype or hiding blockers.
---

## Scope

- `README.md`
- `CONTRIBUTING.md`
- `DOCUMENTATION.md`
- other repo-level documentation normalization work

## Trigger

- when a prompt asks to normalize, tighten, refresh, or de-duplicate documentation
- when root summaries still reflect an earlier phase
- when new authoritative docs need to be linked into the documentation index

## Do Not Use

- Do not use this skill for governance rewrites unless the prompt is actually changing repository law.
- Do not use this skill for implementation-only changes that do not affect documentation state.
