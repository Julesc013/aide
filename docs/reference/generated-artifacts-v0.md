# Generated Artifacts v0

## Purpose

Generated artifacts v0 defines the first deterministic downstream output boundary for AIDE self-hosting guidance.

Generated downstream artifacts help agents consume repo intelligence, but they are not canonical truth. The canonical records remain:

- `.aide/` for the AIDE Profile/Contract.
- `.aide/queue/` for queue execution state.
- `docs/**`, root docs, and `core/**` references for human explanation.
- bootstrap-era records as historical evidence.

## Canonical Source And Generated Targets

Q05 source inputs are small and explicit:

- `.aide/profile.yaml`
- `.aide/toolchain.lock`
- `.aide/commands/catalog.yaml`
- `.aide/queue/policy.yaml`
- `.aide/queue/index.yaml`
- `docs/reference/source-of-truth.md`
- `docs/reference/harness-v0.md`
- `docs/reference/generated-artifacts-v0.md`

Generated targets in Q05 v0:

| Target | Mode | Status |
| --- | --- | --- |
| `AGENTS.md` | managed section | generated by explicit `aide compile --write` |
| `.agents/skills/aide-queue/SKILL.md` | managed section | generated by explicit `aide compile --write` |
| `.agents/skills/aide-execplan/SKILL.md` | managed section | generated by explicit `aide compile --write` |

Final root `CLAUDE.md` and final `.claude/**` remain deferred.

## Marker Format

Managed Markdown sections use deterministic HTML comment markers:

```md
<!-- AIDE-GENERATED:BEGIN section=<section-id> generator=aide-harness-generated-artifacts-v0 version=q05.generated-artifacts.v0 mode=<mode> sources=<comma-separated-paths> fingerprint=sha256:<hex> manual=<policy> -->
...
<!-- AIDE-GENERATED:END section=<section-id> -->
```

The fingerprint is a stable SHA-256 of the generated section body. The manifest records the source fingerprint separately.

## Manifest

Q05 writes `.aide/generated/manifest.yaml`.

The manifest records:

- generator name and version;
- generated manifest schema version, currently `aide.generated-manifest.v0`;
- canonical source paths;
- target path;
- generation mode;
- generated section id;
- manual edit policy;
- source fingerprint;
- generated content fingerprint;
- deferred target posture;
- validation command expectations.

The manifest is deterministic and does not include wall-clock timestamps.

Q06 Compatibility records the generated manifest schema id and generator version as known v0 surfaces. Compatibility checks do not make generated artifacts canonical and do not change Q05 generation behavior.

## Modes

`full-file generated`: not used for final root targets in Q05.

`managed-section generated`: Harness may replace content only between AIDE generated markers. Manual edits are allowed outside markers.

`generated preview only`: Harness may write under `.aide/generated/preview/**`. Preview files are non-canonical and must not be treated as final agent law.

`deferred`: Harness records the target in the manifest but does not create the final file.

## Manual Edit Rules

- Manual edits outside managed sections are allowed.
- Manual edits inside managed sections are review-required and reported as errors by `aide validate` when the section fingerprint no longer matches.
- A generated target without the expected markers is an error.
- Harness refuses to overwrite a markerless preview target.
- Final Claude targets remain absent unless a later reviewed queue item changes policy.

## Stale Output Detection

`py -3 scripts/aide validate` checks generated artifact v0 state after Q05:

- `.aide/generated/manifest.yaml` exists and has the expected schema marker;
- manifest source fingerprint matches current canonical source inputs;
- required managed or preview targets exist;
- generated markers exist;
- generated section body fingerprints match marker fingerprints;
- manifest content fingerprints match generated section fingerprints;
- deferred Claude targets remain non-final.

Q05 v0 treats source-fingerprint drift as a warning/review-required condition and marker/body mismatch as an error.

## Review Expectations

Generated changes must be reviewed like source changes:

- Run `py -3 scripts/aide compile --dry-run` before writing.
- Run `py -3 scripts/aide compile --preview` before final write when Claude preview output is changing.
- Run `py -3 scripts/aide compile --write` only for the approved Q05 target set.
- Run `py -3 scripts/aide validate` after generation.
- Inspect the Git diff for managed section changes and manifest updates.

## Fallback And Recovery

If generation would overwrite markerless content, stop and record a blocker. Do not silently replace manual files.

If a managed section has manual edits inside markers, move the intended manual guidance outside the generated section only through a reviewed edit, then regenerate.

If the manifest is stale, rerun `py -3 scripts/aide compile --write` and review the diff. If the diff is surprising, stop at the review gate.
