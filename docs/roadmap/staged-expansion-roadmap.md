# AIDE Staged Expansion Roadmap

## Purpose

This document keeps later candidate tracks visible without promoting them into current implementation scope.

## First Stack

The first shipped reboot stack is Contract + Harness + Compatibility + Dominium Bridge. Q01 documents that target but does not implement it.

## Deferred Expansion Tracks

- Runtime: future execution substrate after Contract and Harness evidence exists.
- CLI/Service: future surfaces only after queue-approved Runtime or bridge evidence exists.
- Commander: deferred product/control surface, not Q01 scope.
- Mobile: deferred host or app family, not Q01 scope.
- IDE Hosts: future host-family work after the reboot model and compatibility baseline are reviewed.
- Pack/Skill/Workflow IR: candidate intermediate representations for later design and artifact work.
- GStack/reference imports: candidate reference import track after design-mining and compatibility evidence mature.
- Provider adapters and app surfaces: deferred until specific queue items authorize them.

## Promotion Rule

A deferred track becomes active only when a later queue item defines scope, allowed paths, evidence, validation, and review gates. Until then, references to these tracks are future intent, not implemented reality.
