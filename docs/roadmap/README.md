# AIDE Roadmap Documents

## Purpose

This family holds queue-driven reboot roadmap records. It preserves the phase-based root roadmap while giving the reboot its own navigable sequence.

## Records

- [Reboot Roadmap](reboot-roadmap.md): Q00 through Q15 queue sequence and first shipped stack.
- [Queue Roadmap](queue-roadmap.md): operational queue view for current and next reboot items.
- [Staged Expansion Roadmap](staged-expansion-roadmap.md): deferred tracks after the first stack has evidence.

## Boundary

These records are not release schedules. Runtime, CLI or Service surfaces, Commander, Mobile, IDE Hosts, Pack/Skill/Workflow IR, GStack/reference imports, broader release automation, and provider or app surfaces remain deferred unless a later reviewed queue item changes that posture.
