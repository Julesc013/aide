# AIDE Design-Mining Index

## Purpose

Design-mining records are reference inputs for future AIDE design work. They are not AIDE doctrine, not implementation claims, and not support commitments.

Q01 creates only the documentation home and candidate list. It does not deeply research external systems.

## Candidate Records

- [Candidate Systems](candidates.md): systems and ideas that may later produce extracted lessons.

## Boundary

Any lesson extracted here must later be reconciled through AIDE Core, AIDE Hosts, AIDE Bridges, Contract, Harness, Compatibility, Control, SDK, and queue review before it becomes doctrine or implementation.
