# AIDE Decision Records

## Purpose

This family stores concise ADR-like records for reboot decisions. Each record should identify status, context, decision, consequences, and supersession state where relevant.

## Status Vocabulary

Use `accepted`, `proposed`, `superseded`, or `deferred`. Do not mark a decision as implemented unless repository evidence supports that claim.
