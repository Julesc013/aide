# AIDE Vision

## What AIDE Is

AIDE is a long-horizon cross-IDE extension and companion platform for modern and historical development environments. It is intended to combine reusable automation, disciplined repository governance, and host-specific integration work inside one coherent engineering system.

## What AIDE Is Not

- It is not a promise of uniform native parity across every host family.
- It is not a single-host extension presented as a universal platform.
- It is not a license to make compatibility claims before verification exists.
- It is not a substitute for host-specific constraints, extension contracts, or archival realities.

## Why Cross-IDE And Cross-Era

Development environments persist across long time spans, and the useful automation patterns around them do not map cleanly to a single product generation. AIDE exists to handle that reality directly. Some hosts are current. Some are historical but still operational. Some are only accessible through limited extension surfaces or companion workflows. Treating them within one governed project allows consistent metadata, support law, evaluation discipline, and reusable logic without pretending the hosts are identical.

## Shared Core Versus Host Adapters

AIDE is one project with one shared core and many host adapters. The shared core should contain the logic that can be defended as genuinely cross-host: protocols, metadata handling, coordination rules, and other reusable behavior. Host adapters should remain thin and explicit, translating that core into the actual extension or companion mechanism available for a particular host family or compatibility technology.

## Support Honesty

Support honesty is a design requirement, not a documentation afterthought. Different hosts expose different automation surfaces and therefore different ceilings. AIDE must describe support through tiers, states, modes, and capability ceilings rather than by vague supported or unsupported labels. Claims of parity or deep integration are only valid when they are verified for the specific lane being described.

## Agentic Development And Governance

AIDE is designed for long-horizon human plus agentic development. That requires stable repository law, clear naming doctrine, explicit execution logs, and documentation that describes reality. Governance is therefore part of the product engineering system. It exists to keep future implementation work auditable, scoped, and reconstructable as the repository grows across host families and eras.
