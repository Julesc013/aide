# Apple Sources

- Current official: [Xcode overview](https://developer.apple.com/xcode)  
  Used for current Xcode posture, Apple-platform scope, and Apple's current in-IDE coding-assistance messaging.
- Current official: [Xcode support matrix](https://developer.apple.com/support/xcode)  
  Used for current and recent Xcode major-family visibility, macOS host requirements, and high-level Apple platform reach.

## Xcode lineage and release-family evidence

- Current official: [Xcode Release Notes](https://developer.apple.com/documentation/xcode-release-notes/)  
  Used to verify that Apple currently enumerates Xcode 10, 11, 12, 13, 14, 15, 16, and 26 as distinct major release families.
- Archived official: [What's New in Xcode - Archive](https://developer.apple.com/library/archive/documentation/Xcode/Conceptual/WhatsNewXcode-Archive/Articles/Introduction.html)  
  Used to verify archived major-family chapters for Xcode 4, 5, 6, and 7.
- Archived official: [Xcode Release Notes - Archive](https://developer.apple.com/library/archive/documentation/Xcode/Conceptual/RN-Xcode-Archive/Chapters/Introduction.html)  
  Used as the older release-notes archive anchor for pre-current Xcode families.
- Archived official: [Xcode 4 Transition Guide](https://developer.apple.com/library/archive/documentation/IDEs/Conceptual/Xcode4TransitionGuide/Introduction/Introduction.html)  
  Used to verify the Xcode 4 family and the existence of Xcode 3 as the prior major family.
- Archived official: [Using an Existing Xcode 3 Project](https://developer.apple.com/library/archive/documentation/IDEs/Conceptual/Xcode4TransitionGuide/ExistingProject/ExistingProject.html)  
  Used to verify that Xcode 4 reads and builds projects created in Xcode 2.1 through the latest Xcode 3 release.
- Archived official: [64-Bit Transition Guide: Compiling 64-Bit Code](https://developer.apple.com/library/archive/documentation/Darwin/Conceptual/64bitPorting/building/building.html)  
  Used to verify that Apple documentation explicitly refers to Xcode 1.0 and later.
- Archived official: [Technical Note TN2456: Resolving Compatibility Problems on New OS Releases](https://developer.apple.com/library/archive/technotes/tn2456/_index.html)  
  Used to verify Xcode 6, 7, 8, and 9 as recent aligned tool families in Apple's own compatibility guidance.

## Public extension surface and constraints

- Current official: [XcodeKit](https://developer.apple.com/documentation/xcodekit)  
  Used as the canonical Apple API-family anchor for Xcode source editor extensions.
- Current official: [Creating a Source Editor Extension](https://developer.apple.com/documentation/XcodeKit/creating-a-source-editor-extension)  
  Used to verify the public creation flow, the `XCSourceEditorCommand` entry point, Editor-menu invocation, and source-buffer editing scope.
- Archived official: [App Extension Programming Guide](https://developer.apple.com/library/archive/documentation/General/Conceptual/ExtensibilityPG/)  
  Used to verify that Apple treats Xcode Source Editor as a macOS app-extension point.
- Archived official: [App Extensions Increase Your Impact](https://developer.apple.com/library/archive/documentation/General/Conceptual/ExtensibilityPG/)  
  Used for the Xcode Source Editor extension-point listing and Apple app-extension scoping rules.
- Archived official: [Creating an App Extension](https://developer.apple.com/library/archive/documentation/General/Conceptual/ExtensibilityPG/ExtensionCreation.html)  
  Used for containing-app requirements, development-signing realities, App Store and Gatekeeper constraints, and the rule that a containing app must provide user functionality.
