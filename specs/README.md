# AIDE Specs

`specs/` contains product and architecture contracts for AIDE. It defines durable system shape, shared-core boundaries, host-adapter contracts, and schema-oriented behavior models.

This directory is distinct from:

- governance docs, which define repository law and support policy
- research docs, which capture source-backed ecosystem facts
- inventory and matrix files, which record machine-readable ecosystem posture

`specs/` is not an implementation area. It defines contracts that future implementation must follow.
